Lo primero es lo primero, ¿qué es la computación en nube y qué ofrece AWS? Definimos la computación en nube como la entrega bajo demanda de recursos de TI a través de Internet con precios de pago por uso. Esto significa que puede realizar solicitudes de recursos de TI como computación, redes, almacenamiento, análisis u otros tipos de recursos, y luego se ponen a su disposición bajo demanda. No paga por el recurso por adelantado, en su lugar, sólo aprovisiona y paga a final de mes. AWS ofrece servicios y muchas categorías que utiliza conjuntamente para crear sus soluciones. Sólo hemos cubierto unos pocos servicios hasta ahora. Ha aprendido sobre Amazon EC2. Con EC2, puede crear y reducir dinámicamente servidores virtuales llamados instancias EC2. Cuando lanza una instancia EC2, elige la familia de instancias. La familia de instancias determina el hardware en el que se ejecuta la instancia. Puede tener instancias que estén construidas para su propósito específico. Las categorías son propósito general, computación optimizada, memoria optimizada, computación acelerada y almacenamiento optimizado. Puede escalar sus instancias EC2 ya sea verticalmente redimensionando la instancia u horizontalmente lanzando nuevas instancias y añadiéndolas al pool. Puede configurar el escalado horizontal automatizado utilizando Amazon EC2 Auto Scaling. Una vez que haya escalado sus instancias EC2 horizontalmente, necesita algo para distribuir el tráfico entrante entre esas instancias. Aquí es donde entra en juego el Elastic Load Balancer. Las instancias EC2 tienen diferentes modelos de precios. Está el on-demand, que es el más flexible y no tiene contrato. El spot pricing, que le permite utilizar la capacidad no utilizada a un precio con descuento. Planes de ahorro o instancias reservadas, que le permiten suscribir un contrato con AWS para obtener una tarifa con descuento cuando se compromete a un determinado nivel de uso y planes de ahorro que se aplican a AWS Lambda y AWS Fargate, así como a las instancias EC2. También cubrimos los servicios de mensajería. Existe Amazon Simple Queue Service o SQS. Este servicio permite desacoplar los componentes del sistema. Los mensajes permanecen en la cola hasta que se consumen o se eliminan. Amazon Simple Notification Service o SNS se utiliza para enviar mensajes como correos electrónicos, mensajes de texto, notificaciones push o incluso solicitudes HTTP. Una vez que se publica un mensaje, se envía a todos los suscriptores. También ha aprendido que AWS dispone de diferentes tipos de servicios informáticos más allá de los servidores virtuales como EC2. Hay servicios de contenedores como Amazon Elastic Container Service o ECS, y hay Amazon Elastic Kubernetes Service o EKS, ambos de los cuales son herramientas de orquestación de contenedores. Puede utilizar estas herramientas con instancias EC2. Pero si no desea administrar eso, no es necesario. Puede utilizar AWS Fargate, que le permite ejecutar sus contenedores en la parte superior de una plataforma de computación sin servidor. Luego está AWS Lambda, que le permite simplemente subir su código y configurarlo para que se ejecute en base a desencadenantes. Sólo se le cobra cuando el código se está ejecutando realmente, sin contenedores, sin máquinas virtuales, sólo código y configuración. Con suerte, eso lo resume. Nos vemos en la próxima.
