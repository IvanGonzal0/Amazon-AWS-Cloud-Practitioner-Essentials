Hemos estado hablando de algunos recursos diferentes de AWS, así como de la infraestructura global de AWS. Puede que se esté preguntando, ¿cómo interactúo realmente con estos servicios? La respuesta son las API. En AWS, todo es una llamada a una API. Una API es una interfaz de programación de aplicaciones. Lo que esto significa es que existen formas predeterminadas para que usted interactúe con los servicios de AWS. Puede invocar o llamar a estas API para aprovisionar, configurar y administrar sus recursos de AWS. Por ejemplo, puede lanzar una instancia EC2, o puede crear una función AWS Lambda. Cada una de esas serían diferentes solicitudes y diferentes llamadas a las API de AWS. Puede utilizar la consola de administración de AWS, la interfaz de línea de comandos de AWS, los kits de desarrollo de software de AWS, o varias otras herramientas como AWS CloudFormation para crear solicitudes para enviar a las API de AWS para crear y administrar los recursos de AWS. En primer lugar, hablemos de la consola de administración de AWS. La consola de administración de AWS está basada en un navegador. A través de la consola, puede administrar sus recursos de AWS visualmente y de una manera fácil de digerir. Esto es ideal para comenzar y construir su conocimiento de los servicios. También es útil para construir entornos de prueba o ver las facturas de AWS, ver, monitorizar y trabajar con otros recursos no técnicos. Lo más probable es que la consola de administración de AWS sea el primer lugar al que acuda cuando esté aprendiendo sobre AWS. Sin embargo, una vez que esté funcionando en un entorno de tipo de producción, no querrá depender del estilo de apuntar y hacer clic que le ofrece la consola para crear y administrar sus recursos de AWS. Por ejemplo, para crear una instancia de Amazon EC2, tiene que hacer clic a través de varias pantallas, estableciendo todas las configuraciones que desee, y luego lanza su instancia. Si más tarde quisiera lanzar otra instancia EC2, tendría que volver a la consola y hacer clic a través de sus pantallas de nuevo para ponerla en marcha. Al tener que hacer los humanos este aprovisionamiento manual, se está abriendo a errores potenciales. Es bastante fácil olvidarse de marcar una casilla de verificación o escribir mal algo cuando usted está haciendo todo manualmente. La respuesta a este problema es utilizar herramientas que le permitan guionizar o programar las llamadas a la API. Una herramienta que puede utilizar es la interfaz de línea de comandos de AWS, o CLI. La CLI le permite realizar llamadas a la API utilizando el terminal de su máquina. Esto es diferente de el estilo de navegación visual de la consola de administración. Escribir comandos utilizando la CLI hace acciones scriptables y repetibles. Así que puede escribir y ejecutar sus comandos para lanzar una instancia EC2, y si desea lanzar otra, sólo tiene que ejecutar el comando pre-escrito de nuevo. Esto hace que sea menos susceptible al error humano. Puede hacer que estos scripts se ejecuten automáticamente, como en un horario o desencadenados por otro proceso. La automatización es muy importante para tener un despliegue en la nube exitoso y predecible en el tiempo. Otra forma de interactuar con AWS es a través de los Kits de Desarrollo de Software de AWS o SDKs. Los SDKs le permiten interactuar con los recursos de AWS a través de varios lenguajes de programación. Esto facilita a los desarrolladores la creación de programas que utilizan AWS sin utilizar las APIs de bajo nivel, además de evitar esa creación manual de recursos de la que acabamos de hablar.
