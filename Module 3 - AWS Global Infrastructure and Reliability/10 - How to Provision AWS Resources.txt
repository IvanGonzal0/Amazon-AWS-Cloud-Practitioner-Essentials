Muy bien, sigamos hablando de cómo interactuar con AWS. Tiene la consola de administración de AWS, la CLI y los SDK, que son todas formas de hacerlo usted mismo para aprovisionar y administrar su entorno de AWS. Si desea aprovisionar un recurso, puede iniciar sesión en la consola y apuntar y hacer clic, puede escribir algunos comandos, o puede escribir algunos programas para hacerlo. También hay otras formas en las que puede gestionar su entorno de AWS utilizando herramientas gestionadas como AWS Elastic Beanstalk, y AWS CloudFormation. AWS Elastic Beanstalk, es un servicio que le ayuda a aprovisionar entornos basados en Amazon EC2. En lugar de hacer clic en la consola o escribir múltiples comandos para construir su red, instancias EC2, escalado y balanceadores de carga elásticos, en su lugar puede proporcionar el código de su aplicación y configuraciones deseadas al servicio AWS Elastic Beanstalk, que luego toma esa información y construye su entorno para usted. AWS Elastic Beanstalk también hace que sea fácil guardar las configuraciones del entorno, para que puedan ser desplegadas de nuevo fácilmente. AWS Elastic Beanstalk le ofrece la comodidad de no tener que aprovisionar y administrar todas estas piezas por separado sin dejar de ofrecerle la visibilidad y el control de los recursos subyacentes. Usted consigue centrarse en su aplicación empresarial, no en la infraestructura. Otro servicio que puede utilizar para ayudarle a crear implementaciones automatizadas y repetibles es AWS CloudFormation. AWS CloudFormation es una herramienta de infraestructura como código, que le permite definir una amplia variedad de recursos de AWS de forma declarativa utilizando documentos basados en texto JSON o YAML, denominados plantillas CloudFormation. Un formato declarativo como este le permite definir lo que desea construir, sin especificar los detalles de cómo construirlo exactamente. CloudFormation le permite definir lo que desea, y el motor de CloudFormation se preocupará de los detalles sobre cómo llamar a las API para que todo se construya. Tampoco se limita únicamente a soluciones basadas en EC2. CloudFormation admite muchos recursos de AWS diferentes, desde almacenamiento, bases de datos, análisis, aprendizaje automático y mucho más. Una vez que haya definido sus recursos en una plantilla de CloudFormation, CloudFormation analizará la plantilla y comenzará a aprovisionar todos los recursos que haya definido en paralelo. CloudFormation gestiona todas las llamadas a las API back-end de AWS por usted. Puede ejecutar la misma plantilla de CloudFormation y varias cuentas o varias regiones, y creará entornos idénticos en todas ellas. Hay menos margen para el error humano, ya que se trata de un proceso totalmente automatizado. Para recapitular, la consola de administración de AWS es estupenda para aprender y ofrecer una visión al usuario. La consola de administración de AWS es una herramienta manual, por lo que de entrada, no es una gran opción para la automatización. En su lugar, puede utilizar la CLI para programar sus interacciones con AWS utilizando el terminal. Puede utilizar los SDK para escribir programas, para interactuar con AWS por usted, o puede utilizar herramientas administradas como AWS Elastic Beanstalk o AWS CloudFormation.
